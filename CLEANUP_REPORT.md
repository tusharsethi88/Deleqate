# Deleqate — Cleanup Report

> Generated 2026-06-21. Analysis only — see `cleanup.bat` for the executable removals.

## Why a script instead of direct deletion

In this session the workspace's filesystem layer could **create and move** files
(those changes persisted) but could **not delete** them — `rm` either errored or
falsely reported success while the file remained. Rather than report false results,
the safe removals are bundled into `cleanup.bat` for you to run locally.

## Architecture (what's actually live)

Hybrid app: Django serves a React SPA plus some legacy server-rendered pilot pages.
Routing in `backend/config/urls.py` shows:

- `/` and all unmatched routes → `frontend/dist/index.html` (the **built** SPA).
- `/assets`, `/css`, `/img` → served from `frontend/dist/...` (build output, **runtime-required**).
- `/static/...` → served from `static/` (CSS/JS/img for legacy pilot templates).
- `/logo-showroom` → embeds the `static/img/deleqate_*_logos*.png` concept sheets.

## Not attempted: deep dead-code removal

Removing unused Python/JS *functions* on a production app with no git history is risky.
Recommended before any code-level pruning: initialize git (`git init`) for an undo
safety net, then run the app's tests / a smoke test after each change. I can help with
that in a fresh session where deletes work.
